Overview

Port.Scan is a lightweight, self-hosted port scanner. A small Flask backend drives the scanning logic in Python, and this page is served straight from your own machine — no accounts, no telemetry, no external services.

This is a hobby / personal project, not an enterprise product — it's built to be cloned, read, and modified.

Installation

1

Clone the repository.

# replace with your repo URL git clone https://github.com/swe74-29/Python-FlaskWeb-PortScanner.git cd Python-FlaskWeb-PortScanner
2

Create a virtual environment and install dependencies.

python3 -m venv venv source venv/bin/activate # venv\Scripts\activate on Windows pip install -r requirements.txt
3

Run the app.

python app.py # then open http://127.0.0.1:5000 in your browser

Usage

From the home page, fill in:

  • Target host — an IP address or hostname, e.g. 192.168.1.1 or example.com
  • Port range — start and end ports to scan, from 1 to 65535
  • Scan mode — TCP, UDP, or Full (see below)

Click Initiate Scan. Results are written to the scan history table beneath the form.

Only scan hosts and networks you own or have explicit permission to test.

Scan Modes

TCPStandard TCP connect scan — fast, reliable, works everywhere
UDPUDP probe scan — slower, results can be less definitive
FullRuns both TCP and UDP across the given range

Configuration

Environment variables read by the app (placeholder — adjust to your setup):

# .env FLASK_ENV=development SCAN_TIMEOUT=1.5 # seconds per port MAX_THREADS=100 DATABASE_URL=sqlite:///scans.db

Troubleshooting

Permission denied on raw sockets — some scan modes need elevated privileges. Try running with sudo on macOS/Linux.

ModuleNotFoundError — make sure your virtual environment is active before running pip install.

Still stuck? Open an issue on GitHub.
